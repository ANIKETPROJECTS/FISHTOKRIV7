=== FIX 1 — orderId sequence number is wrong ===

The current frontend order got "#FTS202605271" (sequence = 1) but the admin
order from the same day is already "#FTS202605274" (sequence = 4), meaning
there are already 4 orders on May 27.

The sequence counter query is not counting ALL orders for that date — it is
likely only counting frontend (source: "online") orders, or only orders with
a non-null orderId, or using the wrong date boundary.

Fix: the sequence query must count ALL orders in the orders collection where
the orderId starts with "#FTS" + todayDateIST, regardless of source. Example:

  const dateStr = "20260527"; // today in IST, YYYYMMDD
  const count = await Order.countDocuments({
    orderId: { $regex: `^#FTS${dateStr}` }
  });
  const nextSequence = count + 1;
  const orderId = `#FTS${dateStr}${nextSequence}`;

This way both admin and frontend orders share the same sequence pool and
the counter never resets or duplicates.

=== FIX 2 — orderId must be assigned AFTER createdAt/updatedAt ===

Currently orderId is being set before the document is saved (so Mongoose
puts it before createdAt/updatedAt in the stored document).

orderId must be assigned AFTER the initial save, using findByIdAndUpdate,
so it appears as the last field in the document (after createdAt, updatedAt,
and inventoryDeducted). This is the same pattern the admin order uses.

Correct flow:
  1. Build order object WITHOUT orderId
  2. const saved = await new Order(orderData).save()
     --> document now has createdAt, updatedAt set by Mongoose
  3. Generate orderId using IST date + correct sequence count
  4. const final = await Order.findByIdAndUpdate(
       saved._id,
       { $set: { orderId } },
       { new: true }
     )
  5. Return final to client

=== FIX 3 — address string must use · (middle dot) as separator, not comma ===

Admin order address format:
  "205 Kairali Park · Katemanivali, Kalyan · 421306"

Frontend order address format (wrong):
  "205 Kairali Park, Katemanivali, Kalyan, 421306"

When constructing the address string from deliveryAddressDetail fields,
use " · " (space, middle dot U+00B7, space) as the separator between
building, street, and pincode — not a plain comma.

Build it like:
  const address = `${building} · ${street}, ${area} · ${pincode}`;
  // result: "205 Kairali Park · Katemanivali, Kalyan · 421306"
